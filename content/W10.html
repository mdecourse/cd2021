W10

請各組將 Stage2 相關資料繳交到 https://github.com/mdecourse/cd2021/discussions/13

請各組在開始執行 Stage3 專案前, 將 Stage1 與 Stage2 各組員所參與執行的專案內容, 寫在 Stage3 的分組網站上.

Stage3 將在 ForkIT Park 的說明下開始進行. 其中園區將包含機電資整合遊樂區與自動化工廠展示區.

採用 Onshape, Github 與 Coppeliasim (或 Webots) 的協同產品設計特性:

利用 MCAD (Mechanical Computer Aided Design) 套件進行設計繪圖後, 將零組件轉入機電整合系統模擬過程, 除了納入感測器與監控系統外, 還必須根據 Kinematic 模擬確定機械構件的組合搭配可以滿足設計需求, 並且就剛體零組件的 Dynamic 分析決定控制系統參數設計以及致動器 (Actuator) 的容量設計等. 採用可遠端監視遙控並進行接近實體模型運作流程的的虛擬展示, 可有效提升協同產品設計的效能.

本課程所完成的協同產品設計模型, 必須能夠採 3D 列印的方式完成大多數零件, 並且搭配鋁擠件,  Arduino 或Raspberry Pi 控制卡, 各式電子感測器 (或極限開關), 直流馬達, 伺服馬達與線性滑軌 (或光軸) 等元件配合各式機電標準零組件製作實體模型.

而利用分散式版次管理的流程管理設計文案, 可以讓各參與協同的人員明確註記各設計階段進行時, 所完成的設計改版內容. 必要情況下, 各設計者可以從設計倉儲中 branch 出其他設計修改, 不致影響主分支的既定設計流程, 待其他設計分支內容經過討論與驗證後, 再決定是否與主分支內容進行合併.
