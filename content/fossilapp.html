fossilapp

fossilapp 是希望能夠用於 Fossil SCM 用戶註冊帳號時, 提供網際表單讓使用者自行設定帳號密碼並完成能夠展開 Github 倉儲靜態網頁的設定.

為了在遠端電腦上以 SSH 連線到 Fossil SCM 主機進行 fossilapp 網際程式的開發, 因此啟動 Xming, 並設定 putty.exe session, 以 X11 forwarding (X display location 設為 localhost:0.0) 將 X-windows 傳送到 Windows, 首先以 sudo pip3 install leo 安裝 Leo Editor.

Leo Editor 安裝後, 就可以透過 leo& 在 background 開啟至近端, 接下來就可以進行 fossilapp 的開發.

第一版的 fossilapp 先將表單與處理表單變數的函式架構先建起來:

fossilapp.py

from flask import Flask, request

# for generating secret_key
import os

app = Flask(__name__)

# 使用 session 必須要設定 secret_key
# In order to use sessions you have to set a secret key
# set the secret key.  keep this really secret:
secret_key = os.urandom(24).hex()
app.secret_key = secret_key

@app.route("/")
def hello():
    return "<h1 style='color:blue'>Hello There!</h1>"

@app.route('/form')
def form():
    """Create form routine"""
    
    return "<html><body><h1>Create Fossil SCM Repository</h1><form method='post' action='genAccount'> \
     Account:<input type='text' name='account'><br \> \
     Password:<input type='password' name='password'><br \> \
    <input type='submit' value='genAccount'></form> \
    </section></div></body></html>"
    
@app.route('/genAccount', methods=['POST'])
def genAccount():

    """Generate Fossil SCM account
    """
    # get user input account and password
    account = request.form["account"]
    password = request.form["password"]
    return "account:" + account+"<br />" \
    + "password:" + password
if __name__ == "__main__":
    app.run(host='0.0.0.0')

wsgi.py

import fossilapp

uwsgi = True

application = fossilapp.app

if __name__ == "__main__":
    
    if uwsgi:
        application = fossilapp.app
    else:
        fossilapp.app.run()

第一版的 fossilapp 網際程式僅在 /form 執行時列出帳號與密碼表單, 送出後由伺服器確定已經取到此兩個輸入變數.

而每次遠端 fossilapp 程式修改後, uwsgi 對應的 Flask wsgi 服務就必須重新啟動, 以便納入程式碼的改變 (若在近端以 fossilapp.app.run() 執行, 程式碼修改存檔後, 則 Flask 會自動重新啟動, 無需手動重啟).

重新啟動 /etc/systemd/system/wsgi.service 服務, 執行:

sudo systemctl restart wsgi

上述流程所建立的 Flask 網際程式原始碼已經利用 git init 將程式目錄設為 Git 倉儲, 並且透過下列指令, 以 main 分支送到 https://github.com/mdecourse/fossilapp.

新增 .gitignore, README.md 與 LICENSE.

並且執行:

git add .
git commit -m "initial add"
git branch -M main
git remote add origin https://github.com/mdecourse/fossilapp.git
git push -u origin main

git add . 表示要將所有新增的內容進行 tracking, 即拍一個現存所有內容的快照. 接著的 git commit 提交則以 "initial add" 提交字串進行註記, 說明這是將 fossil app 程式碼提交到 Git 倉儲的第一個新增版本.

git branch -M 中的 -M 是 --move --force 指令選項的縮寫, 因此 git branch -M main 的指令是要求將目前倉儲的內容強制轉為名稱為 main 的分支.

至於 git push -u origin main, 指令中的 -u 是 --set-upstream 的縮寫, 在 git push 後加上 -u origin main 表示目前的 push 與隨後的推送, 在不指定 url 名稱與分支的情況下, 其 tracking 的 url 內定為 origin, 而分支則內定為 main.

以下為已經可以接受使用者輸入帳號與密碼, 然後建立對應 Fossil SCM 倉儲的 fossilapp 網際程式第二版本:

Fossil SCM 建立帳號流程:

接下來的 fossilappa 網際程式的開發, 主要牽涉 Fossil SCM 倉儲的建立, 由使用者利用 https 表單輸入建立倉儲後要使用的帳號與密碼後, 即交由系統建立對應帳號, 並利用 Fossil SCM 的指令建立對應密碼與倉儲權限.

此一可用網際 fossilapp 程式的版本位於 fossilapp.py.

基本程式流程:

template.fossil 已經設定可讓 Fossil SCM 以 Documentation 功能展開 Github Pages 靜態網頁的選項, 其中包含:

  1. default-csp: https://fonts.googleapis.com
  2. ignore-glob: .DS_Store 與 __pycache__
  3. 勾選 dotfiles

利用 fossil user new 指令修改上述步驟所複製的 fossil 倉儲對應使用者的管理密碼.

利用 fossil user capabilities 指令修改 fossil 倉儲對應使用者帳號的管理權限 (讓使用者升級為管理者).

利用 fossil user capabilities 指令修改 fossil 倉儲 template 對應帳號的管理權限 (刪除 cd 帳號管理權限).

防止 Shell Command Injection:

由於目前的網際程式透過表單取使用者的帳號與密碼, 並利用此資料直接執行 shell commands, 因此為了防範使用者利用輸入字串執行 shell commands, 因此帳號只允許輸入數字 (即學員學號), 而密碼則排除可用於 shell commands 的特殊字元, 初步的輸入字串判定程序為:

    # To avoid shell command injection, accept only numbers for Account
    if account.isdigit():
        pass
    else:
        return "Accept only numbers for Account!"
    
    # To avoid shell command injection, no special characters allowed for Password
    for i in range(len(password)):
        if password[i] in ["&", "|", ";", "$",">", "<", "`", "\\","!"]:
            return "No special characters allowed for Password!"

接下來為了更進一步管制使用本 fossilapp 網際程式的族群, 只限縮於 KMOLab 本學期選課的學員, 且只限於目前 Github 帳號被 flagged 的學員, 因此必須置入 Google Oauth2 的登入管控機制, 除了只允許擁有 @gm 電子郵箱的用戶登入外, 並且限制只有帳號被進用的學員登入.

另外一個處理方法則是透過網際表單, 讓 Github 帳號被禁用的學員至 fossilapp 登記時, 由系統產生一串編碼的連結, 以 email 寄出, 而使用者點擊回網際系統時再直接取使用者的學號, 並由使用者選擇要管理 Fossil SCM 的密碼後, 再完成對應 Fossil SCM 倉儲的建立.

在此先選擇採 Google Oauth2 的方式建立登入管理系統.
