account flagged

Github account flagged 的解決方法:

  1. 寫信詢問為何會被封鎖, 並尋求解除封鎖.
  2. 自行建立 gogs 與 Fossil SCM 伺服器, 利用 gogs 展開原先在 Github 倉儲中的歷程資料, 利用 Fossil SCM 伺服原先在 Github Pages 上的靜態網頁.

gogs 主機可以透過 https://letsencrypt.org/ 取得公開認證的數位簽章, 若使用自行簽署的數位簽章, 則用戶必須在執行 git 指令之前將 http.sslVerify 設為 false: git config --global http.sslVerify false

建立 gogs 與 Fossil SCM 伺服器:

以下為利用 Stunnel 建立 TLS/SSL tunneling service for gogs 與 Fossil SCM servers 所需的設定檔案:

[https]
accept = server_domain_name:443
cert = /etc/stunnel/localhost.crt
key = /etc/stunnel/localhost.key
exec = /usr/bin/fossil
execargs = /usr/bin/fossil http /home/user_account/repository/ --https --nojail --notfound cd2021

[https]
accept = server_domain_name:3443
connect = server_domain_name:3000
cert = /etc/stunnel/localhost.crt
key = /etc/stunnel/localhost.key

其中第一段, 將 port 443 設為 Fossil SCM 的網際連線埠號, 而 localhost.crt 與 localhost.key 則取自 /etc/letsencrypt/live/server_domain_name/fullchain.pem 與 /etc/letsencrypt/live/server_domain_name/privkey.pem.

至於 Fossil SCM 的用戶倉儲目錄設為 /home/user_account/repository, 採用 https 啟動, 且以 drop root privilege but do not enter the chroot jail 的模式啟動, 並且內定倉儲設為 cd2021.fossil.

而第二段則將 gogs 的對外 https 連線埠號設為 3443, 而 http 連線埠號則為 3000, 啟動時也是採用上述 Letsencrypt 所取得的公開認證數位簽章檔案.

由於 gogs 具備使用者 register 功能, 因此 Github 帳號被 flagged 的使用者可以自行至 gogs 網站註冊帳號後, 即可將原先 Github 倉儲同步至 gogs 同名倉儲. 但是若要讓每一位 Github 帳號被 flagged 的使用者都建立一個 Fossil SCM 倉儲則可以利用 Flask 編寫一個網際程式, 讓使用者以 @gm 登入系統後, 直接取用學號建立對應的 Fossil SCM 倉儲, 並設法完成能夠展開 CMSiMDE 靜態網頁的環境設定.

為了建立此一網際服務, 必須依照 http://mde.tw/wcm2021/content/W11.html 的步驟, 以 uwsgi 與 nginx 提供一台應用程式執行伺服器.

Python 應用程式伺服器設定:

以 pip3 安裝 flask 與 flask_cors

sudo pip3 install flask flask_cors

接下來安裝 uwsgi 所需程式庫與模組:

sudo apt install uwsgi uwsgi-plugin-python3

sudo pip3 install uwsgi

Nginx 設定:

設定目的在利用 port 9443 建立 https 連線伺服器, 並且從 8080 埠號執行的 uwsgi Flask 程式取得網際資料.

# for creating web-based flask application server
server {
    listen 9443 ssl;
    listen [::]:9443 ssl;
    
    location / {
        include uwsgi_params;
        uwsgi_pass  127.0.0.1:8080;
    }
    
    ssl_certificate /etc/stunnel/localhost.crt;
    ssl_certificate_key /etc/stunnel/localhost.key;
    ssl_session_timeout 5m;
    ssl_protocols SSLv3 TLSv1 TLSv1.1 TLSv1.2;
    ssl_ciphers "HIGH:!aNULL:!MD5 or HIGH:!aNULL:!MD5:!3DES";
    ssl_prefer_server_ciphers on;
    try_files $uri $uri/ =404;
}

uwsgi 設定:

位於 uswsig_ini 目錄中的 uwsgi.ini 則宣告所要執行的 uwsgi 程式位置.

[uwsgi]
socket = :8080
uid = user_account
gid = user_account
plugins-dir = /usr/lib/uwsgi/plugins/
plugin = python3
master = true
process = 4
threads = 2
chdir = /home/user_account/github/fossilapp
wsgi-file = /home/user_account/github/fossilapp/wsgi.py

最簡單的網際 Flask 程式:

fossilapp.py

from flask import Flask
app = Flask(__name__)

@app.route("/")
def simplest():
    return "<h1>The Simplest Flask Web-based Program!</h1>"

if __name__ == "__main__":
    app.run(host='0.0.0.0')

wsgi.py

import fossilapp

# 若 True 則以 uwsgi 模式執行
uwsgi = True

application = fossilapp.app

if __name__ == "__main__":
    if uwsgi:
        application = fossilapp.app
    else:
        fossilapp.app.run()

最後為了要讓 uwsgi 程式以服務執行, 必須設定 /etc/systemd/system/wsgi.service 檔案.

[Unit]
Description=uWSGI to serve fossilapp
After=network.target
   
[Service]
User=user_account
Group=user_account
WorkingDirectory=/home/user_account/uwsgi_ini
ExecStart=/usr/bin/uwsgi --emperor /home/user_account/uwsgi_ini
   
[Install]
WantedBy=multi-user.target

設定完成後, enable wsgi 服務, uwsgi 程式會隨開機程序啟動執行:

sudo systemctl enable wsgi

啟動, 關閉及重啟 wsgi 服務:

sudo systemctl start wsgi

sudo systemctl stop wsgi

sudo systemctl restart wsgi

而 uwsgi 的 https 則是由 Nginx 提供, 啟動, 關閉或重啟服務:

sudo /etc/init.d/nginx start

sudo /etc/init.d/nginx stop

sudo /etc/init.d/nginx restart

由於 gogs 與 Fossil SCM 都是依賴 Stunnel 啟動, 開關或重啟服務:

sudo /etc/init.d/stunnel4 start

sudo /etc/init.d/stunnel4 stop

sudo /etc/initld/stunnel4 restart

接下來為了利用 Fossil SCM template 倉儲, 從網際表單讓使用者自行登錄建立能夠展開 Github 倉儲 Pages 靜態網頁的系統, 必須參考 http://mde.tw/wcm2021/content/Fossil%20scm.html 與 Google oauth2 介面, 讓使用者登入系統後填表建立對應的 Fossil SCM 倉儲, 之後再 fossil clone 至近端與登錄至 gogs 倉儲的檔案整合, 最後再 fossil commit, 即可將整合後的倉儲內容推向 Fossil SCM server, 並在 Fossil Documentation 的機制下, 伺服整合至 Fossil SCM 之後的各版本的靜態網頁內容.
