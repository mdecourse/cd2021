W14-W15

CoppeliaSim 控制程式學習流程:

  1. Lua embedded scripts - 根據官方簡介與 Tutorial, 在網際解譯器中執行, 並參考 CAD2020 CoppeliaSim 頁面中的範例, 了解如何利用 non-threaded and threaded Lua 操控模擬場景中的機電系統.
  2. Legacy Python remote API - 根據簡介, 客戶端需求說明與 Tutorial, 並參考 CAD2020 CoppeliaSim 頁面中的範例, 查詢 Python remote API 函式, 了解如何利用 Python 操控模擬場景中的機電系統.
  3. B0-based remote API - 根據簡介了解如何利用 Python 操控模擬場景中的機電系統.
  4. 在 Python remote API 中導入 Flask, 並啟動 server 的多埠號架構, 將場景中的 image sensor 影像串流至各操控端瀏覽器, 以便執行組員間的場景協同控制系統設計.

機械手臂取放物件專案學習流程:

  1. 自行利用 SolidWorks, Inventor, NX, Onshape 或 Solvespace 繪製機械手臂零組件後, 分別以 ASCII STL 或 Binary STL 手動轉入 Coppeliasim 建立機械手臂 .ttm 模型備用.
  2. 根據 CoppeliaSim 4.1.0 版 (4.2.0 無法開啟 .simplescene.xml 檔案) 中的 XML simplescene 格式, 利用 Python 程式建立所需的機械手臂 .ttm 模型備用.
  3. 根據 RoKiSim 中的六軸機械手臂定義, 以 Python 程式, 將既有的 Industrial Robots STL 轉為 CoppeliaSim simplexml 模型格式備用.
  4. 將上述完成的機械手臂模型導入場景後, 機械手臂末端利用 force sensor 接上 components/grippers/suction pad.ttm tool 後, 先完成 MTB Robot 取放 Block 範例的改寫, 再完成 FANUC M-710iC/50 機械手臂的圓球取放範例.

協同產品設計實習課程, W14 週的取分項目檔案位於: MTB Robot 取放 Block 控制.

背景說明:

所謂的 MTB Robot 是 https://www.coppeliarobotics.com/helpFiles/en/robotLanguageIntegrationTutorial.htm 中用來展示其 plugin 功能, 可以自行設計機械手臂操控語法, 並編寫客製化解譯 plugin 來解讀為 CoppeliaSim 操控流程.

取分項目:

  1. 請簡化上列四軸機械手臂取放方塊零件的 Lua 程式, 去除客製化控制 plugin 語法部分, 且刪除其舊有的 GUI 程式控制介面, 改用 CoppeliaSim 制式的 Lua Script 編寫與原先取放方塊零件相同的操控流程.
  2. 請採用 CoppeliaSim Qt based GUI 介面, 改寫原 MTB robot.ttm 模型中的 legacy GUI 介面視窗.

第十五週取分項目:

  1. 請分別根據下列兩部影片, 整理出影片字幕, 放入個人與分組網站標題為 W15 的頁面中.
  2. 請利用上述 Inverse Kinematic 運算, 計算各軸轉動角度後, 以 Lua Script 控制 MTB Robot, 令其將方塊中心在 (0.2, 0.7, 0.05) 與 (-0.3, -0.55, 0.05) 等兩個位置之間取放. 完成後, 請將程式碼附上說明連同 Robot 取放過程影片與場景檔案, 放入個人與分組網站標題為 W15 的頁面中.
