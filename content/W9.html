W9

期中考週流程:

  1. 利用 ::17 開啟 Ethercalc 表單, 收集各班學員課程期望分數, 將期中期望成績送至教務主機.
  2. 此一 W1-W9 各學員的期望成績, 將與學員的個人與分組倉儲內容進行比較後, 會反應至學員的期末學期成績中, 若實際評量結果與學員期望成績產生一定差距, 將酌予加減各學員應得之課程學期成績.
  3. 各分組進行 5-10 分鐘的網際 stage2 分組專案報告, 並透過 ShareX 錄製簡報影片 (簡報前後按下 Shift + PrintScreen, 並採全畫面錄影), 之後會將各組簡報影片送到 GDrive 公開展示, W15 之前將要求各組針對 stage1 與 stage2 的其他組別專案進行互評, 以供實際評分參考.

課程衍生議題尋求解決方案 (各學員取分途徑):

計算機程式, 網際內容管理, 電腦輔助設計實習與協同產品設計實習的目的, 在透過數位與網路方法提升技職體技機械設計工程相關學生的競爭力, 其中主要面向在尋求以學員協同流程, 建立虛擬機電資整合環境 (也稱為 ForkIT 機電資整合模擬園區).

取分議題 1:

ForkIT Park 頁面希望整理 CD2021 stage1 與 stage2 各組簡報影片的連結與零組件機電模型, 並針對各組已經完成的內容進行延伸建議, 最後目的希望可以將所完成的 CAD 模型與相關程式整合進 ForkIT 園區 (亦即, 在 Stage3 專案期間, 各組要將各組員在 Stage1 與 Stage2 所完成的內容或先前所完成的模型整合至 Stage3 模擬場景中, 並配合 Python Remote API 遠端程式進行監控).

目前的 ForkIT Park 頁面只完成 ag1-ag4 的資料整理, 在此分別徵求各組採 Pull Request 方式完成兩班的 stage1 簡報影片連結, 並在自己的帳號下展示已經完成的 eng fork 網站內容.

取分議題 2:

請各分組在執行各 stage 專案內容時, 針對 ForkIT Park 所提到的相關技術中, 將所完成的機電資整合系統納入 ForIK Park 園區 (即 CD2021 各班小組所完成的模型項目與環境總集合) 進行模擬, 並就建模, 組立, 轉檔, 建立場景, 編寫程式等流程進行詳細說明, 例如:

1. 建立 CAD 零組件時採用不同套件時, 從軟體取得, 安裝, 配置到協同產生可用零組件的過程, 分別面臨那些問題? 如何克服? 能否在不同階段, 展示各組員分工流程圖與目標達成比例等.

2. 哪一組可以提供各種不同 CAD 在建立機電資整合專案零組件的工程, 簡潔有用的 CAD 操作 tutorial 教學方案 (例如: 對 https://learn.onshape.com/collections/onshape-fundamentals-cad 或 http://www.farwire.net/SolveSpace-LearningGuide.htm 內容進行摘要整理, 教導其他從來沒有用過 Onshape 或 Solvespace 的學員快速學會用於建立所需零組件).

3. 將 CAD 零組件轉入 CoppeliaSim 過程, 場景中是應該直接利用外部零組件進行模擬運算, 或者如套件手冊建議, 設法從複雜零件 (展示用), 建立簡化 (運算用) 零組件後, 再利用 hide and show 設定進行模擬.

4. 當模擬園區中充滿機械手臂, 推高機, 輸送帶以及各種 STEM (Science, Technology, Engineering and Mathematics) 遊樂展示設施時, 是否應該建置許多安全感測器, 並導入 Human 模型, 讓園區可以更貼近實際的製造工廠或遊樂園區?

5. CoppeliaSim 於 2021.04.6 推出 4.2.0 版 (可攜版下載 142MB), 對先前所建立的模型有何影響, 有哪些新功能可以使用?

6. 哪一組可以整理 https://www.coppeliarobotics.com/resources 中的資料, 或者將所面臨的問題, 至 https://forum.coppeliarobotics.com/ 提問, 並得到可用的解決方案?

7. W15 之前將要求各組針對 stage1 與 stage2 的其他組別專案進行互評, 該如何進行評分? 如何取分? 可用的程式套件在哪裡?
